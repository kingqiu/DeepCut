# DeepCut - 技术规格说明书 (SPEC)

> 最后更新：2026-02-28 | 版本：v0.2（需求沟通后重写）

## 1. 产品概述

DeepCut 是一个智能短视频切片系统，通过多模态 AI 分析（语音识别 + 场景检测 + 运镜变化），将长视频自动切割为 5~30 秒的细粒度短视频片段。每个片段作为独立单元存入全局片段库，并附带多维度 AI 标签。未来 AI 可根据用户指定的主题，从片段库中自动选片、组合、添加字幕/配乐/转场，生成可发布到抖音/小红书的成品短视频。

### 产品愿景（三步走）

| 阶段 | 目标 | 交付物 |
|------|------|--------|
| **MVP** | 切片引擎 | Python CLI：本地视频 → 细粒度切片 + 多维度标签 |
| **第二阶段** | Web 平台 | 上传/链接获取 + 管理页面 + 预览下载 |
| **远期** | AI 重组 | 用户给主题 → AI 自动选片+拼接+字幕/配乐/转场 → 多个成品 |

### 核心用户场景

1. **内容创作者**：博主探店长视频 → 自动切成数十个 5~30s 片段 → AI 打标签 → 后续 AI 重组为抖音/小红书短视频
2. **媒体运营**：从访谈、会议、直播回放中自动提取关键片段
3. **自媒体矩阵**：跨视频片段库统一管理，AI 自动组合生成多条不同风格的短视频

## 2. MVP：切片引擎

### 2.1 输入规格

| 项目 | 规格 |
|------|------|
| 输入方式 | 本地文件路径（CLI 参数） |
| 视频格式 | FFmpeg 能解码的所有格式（MP4, MOV, AVI, MKV, WebM 等） |
| 视频时长 | 5~30 分钟（后续扩展到 1 小时+） |
| 画幅方向 | 横屏 (16:9) 和竖屏 (9:16) 都支持 |
| 编码 | H.264, H.265/HEVC, AV1 等 |
| 语言 | 中文、英文、中英混合 |

### 2.2 用户可配置参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--input` | 视频文件路径（必填） | - |
| `--duration-preference` | 时长偏好：`short`(5-10s) / `medium`(10-20s) / `long`(20-30s) | `medium` |
| `--detectors` | 启用的检测维度：`scene,speech,motion` | 全部启用 |
| `--output-dir` | 输出目录 | 视频所在目录下自动建子目录 |

### 2.5 AI 标签体系

每个切片附带多维度标签，为后续 AI 重组提供依据：

| 维度 | 说明 | 示例值 |
|------|------|--------|
| **content** (内容) | 画面/场景内容描述 | 室内、室外、美食、产品、人物、风景 |
| **emotion** (情绪) | 片段情绪氛围 | 兴奋、平静、搞笑、紧张、温馨 |
| **technical** (技术) | 拍摄技术特征 | 固定镜头、平移、推拉、航拍、横屏、竖屏 |
| **purpose** (用途) | 适合的使用场景 | 适合开头、适合结尾、高能片段、转场素材 |

> 标签清单将在开发过程中逐步完善，初始版本可由 LLM 自由生成，后续收敛为预定义清单。

### 2.6 切片优先级策略

| 场景 | 主切分依据 | 辅助依据 |
|------|-----------|----------|
| **有人声视频** | 语音语义边界（完整句/话题变化点） | 场景检测 + 运镜变化 |
| **无人声视频** | 场景切换检测 | 运镜变化 + 光流分析 |
| **混合场景** | 有人声段落→语义优先；无声段落→视觉优先 | 自动切换策略 |

**核心原则**：
- 人声检测 (VAD) 区分"有人声"和"无人声"，而非简单判断音频轨是否存在
- 5~30 秒是参考范围，不是硬限制，系统根据内容自然边界智能判断
- 每个片段应是一个语义完整的最小单元

## 3. 第二阶段：Web 平台（MVP 后）

### 3.1 视频输入

| 功能 | 说明 |
|------|------|
| Web 上传 | 拖拽上传，分片上传，断点续传（参考 AutoClip 界面） |
| YouTube 链接 | 粘贴链接自动下载视频（yt-dlp） |

### 3.2 管理功能

| 功能 | 说明 |
|------|------|
| 视频列表 | 查看已上传视频、处理状态 |
| 切片预览 | 在线播放切片、查看标签 |
| 缩略图 | 每个切片自动抽取代表帧作为封面 |
| 时间线 | 可视化标注切片区间 |
| 下载 | 单片下载、ZIP 批量打包下载 |

### 3.3 全局片段库

所有视频的切片统一进入全局片段库：
- 跨视频搜索、按标签筛选
- 多维度标签聚合视图
- 为 AI 重组提供素材池

## 4. 远期：AI 重组引擎

### 4.1 核心能力

| 能力 | 说明 |
|------|------|
| 自动选片 | 用户给主题/方向 → AI 从全局片段库中选择匹配片段 |
| 智能排序 | AI 决定片段顺序，保证叙事连贯 |
| 拼接合成 | 自动拼接、添加转场效果 |
| 字幕生成 | 基于转录文本自动生成字幕 |
| 配乐 | 自动匹配背景音乐 |
| 片头片尾 | 自动添加 |
| 批量输出 | 一次生成多个短视频（数量由用户指定） |
| **横转竖** | 横屏片段自动转为竖屏输出（智能裁切/主体居中/模糊填充） |

### 4.2 输出目标

- 格式：竖屏 (9:16) 为主（抖音/小红书）
- 时长：15s / 30s / 60s（适配不同平台要求）
- 成品可直接发布，无需人工二次编辑

## 5. 数据模型（MVP 阶段为 JSON 文件，Web 阶段迁移到数据库）

### MVP 阶段：JSON 文件

切片引擎的输入输出完全基于文件系统，不依赖数据库。元数据以 JSON 格式存储（详见 §2.4）。

### Web 阶段：PostgreSQL + Prisma

```
Video (源视频)
├── id, path, title, duration, resolution, orientation, codec, has_speech
├── file_size, mime_type, status
├── created_at, updated_at
│
├── ClipVersion (切片版本)
│   ├── id, video_id, version_tag, config_json
│   ├── created_at
│   │
│   └── Clip (切片)
│       ├── id, version_id, video_id
│       ├── sequence_index, scene_group
│       ├── start_time, end_time, duration
│       ├── file_path, thumbnail_path
│       ├── split_reason, overlap_prev, overlap_next
│       ├── transcript_segment
│       │
│       └── ClipTag (标签 - 多维度)
│           ├── id, clip_id, dimension, value
│           └── (dimension: content|emotion|technical|purpose)
│
├── Transcript (转录文本 - 持久化)
│   ├── id, video_id, language
│   ├── segments: [{start, end, text}, ...]
│   └── created_at
│
└── ClipRelationship (切片关系)
    ├── id, clip_id, related_clip_id
    ├── relationship_type (sequence|same_scene|context_continuation)
    └── metadata_json
```

### 全局片段库

```
GlobalClipLibrary (全局视图，跨视频)
├── clip_id → 指向 Clip
├── video_id → 来源视频
├── 所有标签维度（可搜索/筛选）
├── orientation (横屏/竖屏)
└── 用于 AI 重组的向量索引（未来）
```

## 6. 非功能性需求

| 维度 | MVP | Web 阶段 |
|------|-----|----------|
| **性能** | 处理时间 < 视频时长 × 3 | < 视频时长 × 2 |
| **视频时长** | 5~30 分钟 | 扩展到 1 小时+ |
| **并发** | 单任务 | 支持 10 个并行任务 |
| **可靠性** | 日志记录，失败可重跑 | 自动重试 3 次 + 死信队列 |
| **可观测性** | 终端日志打印进度 | 全链路追踪 + AI 调用指标 |

| 首页 = 输入 + 项目列表 | ✅ 采纳此布局模式 | DeepCut 增加全局片段库入口 |
| 链接导入 (B站/YouTube) | ✅ 采纳 | 用 yt-dlp 封装 |
| 文件拖拽上传 | ✅ 采纳 | DeepCut 不需要 SRT 上传（自带 Whisper） |
| 视频分类选择 | ❌ 不采纳 | DeepCut 用 VAD 自动判断，无需手动分类 |
| 项目卡片 | ✅ 采纳卡片形式 | DeepCut 增加标签预览、画幅方向标记 |
| 项目详情页 = 切片展示 | ✅ 采纳此布局 | DeepCut 增加多维度标签筛选、片段关系可视化 |
| AI 合集（横向滚动） | ⚠️ 远期参考 | DeepCut AI 重组引擎更强大（全局片段库选片） |
| 切片卡片（缩略图+标题+摘要+操作） | ✅ 采纳此交互模式 | DeepCut 增加标签展示、画幅标记 |
| 评分排序 | ❌ MVP 不采纳 | DeepCut MVP 不评分，后续可加 |
| 投稿到 B站 | ❌ 暂不采纳 | 远期可考虑 |
| 全局片段库浏览 | 🆕 DeepCut 新增 | AutoClip 无此功能，DeepCut 核心需求 |

### AutoClip Pipeline 对比

| 步骤 | AutoClip | DeepCut |
|------|----------|---------|
| 切分依据 | 纯语音字幕 (SRT) | 多模态：语音 + 场景 + 运镜 |
| 切分粒度 | 按话题（1-5 分钟） | 细粒度（5-30 秒） |
| 无声视频 | ❌ 不支持 | ✅ 纯视觉方案 |
| 标签 | 单维度 | 多维度体系 |
| 片段关系 | 无 | 记录时序/场景/上下文 |
